# ButcherCraft Asset Specifications

Status: v0.8.0E packaging asset handoff contract

## Scope

This document defines technical requirements for the current packaging presentation assets. It does not define final artwork. Final textures should follow the ButcherCraft Asset Bible: industrial, functional, authentic, clean, professional, and compatible with vanilla Minecraft proportions.

## Shared Rules

- Ordinary item and block-face textures must be 16x16 PNG unless a future art review approves 32x32 for readability.
- GUI textures may use power-of-two canvases; the Packaging Table GUI uses a 256x256 PNG with a 176x166 active region.
- Use normal Minecraft resource locations so resource packs can replace artwork without code changes.
- Do not use animated textures, emissive layers, connected textures, custom renderers, or dynamic labels for these assets.
- Current placeholder files are intentionally unfinished and should be replaced in place.

## Display Transform Policy

All current packaging supply and retail packaging items intentionally use `minecraft:item/generated`. No explicit display transform overrides are present because inherited generated-item transforms are sufficient for flat inventory sprites. Add transforms only after client review proves a specific item is clipped, too small, too large, or unreadable in one of the standard contexts.

## Placeholder Policy

Placeholder textures use 16x16 PNG files at the final replacement locations. They avoid the vanilla missing-texture pattern, load through normal resource locations, and are documented as `Placeholder` in `docs/ASSET_MANIFEST.md`. Replacing a placeholder must not require code, registry, or datapack changes.
